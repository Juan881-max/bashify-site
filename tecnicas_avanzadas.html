Técnicas Avanzadas en Bash Scripting

Una vez dominados los conceptos básicos, puedes explorar técnicas más potentes para resolver problemas complejos y crear scripts robustos.

Expresiones Regulares (Regex)

Las expresiones regulares son patrones para buscar y manipular texto. Bash las utiliza a menudo junto con herramientas como grep, sed y awk, o en condiciones avanzadas con [[ ... =~ ... ]].

#!/bin/bash

texto="Bash scripting es genial, la version es 5.0"

# Buscar líneas que contienen "scripting" (ignorando mayúsculas/minúsculas)
echo "$texto" | grep -i "scripting"

# Buscar líneas que contienen un número de versión (patrón d.d)
echo "$texto" | grep -E "[0-9]+\.[0-9]+" # -E for extended regex

# Usando [[ ... =~ ... ]] en una condición
if [[ "$texto" =~ [0-9]+\.[0-9]+ ]]; then
    echo "El texto contiene un número de versión."
fi

Manejo de Errores

Es crucial que los scripts manejen errores de forma elegante para evitar comportamientos inesperados o fallos silenciosos.

  • set -e: Sale inmediatamente si un comando termina con un código de salida distinto de cero.
  • set -u: Trata las variables no definidas como un error.
  • trap 'comandos' SEÑAL: Ejecuta comandos cuando recibe una señal (ej. EXIT, ERR, INT para interrupción).
#!/bin/bash

# Sale si cualquier comando falla
set -e
# Sale si usas una variable no definida
set -u

# Define una función que se ejecutará al salir (éxito o error)
cleanup() {
  echo "Limpieza terminada."
  # Puedes añadir comandos de limpieza aquí
}
trap cleanup EXIT

# Define una función que se ejecutará si ocurre un error
handle_error() {
  echo "Error detectado en línea $LINENO."
  # Puedes añadir lógica para manejar errores específicos
}
trap handle_error ERR


echo "Inicio del script."

# Este comando fallará si el directorio no existe y set -e está activo
# rm /directorio/que/no/existe/archivo.txt

# Este comando usará una variable no definida y fallará con set -u
# echo "Variable: $variable_no_definida"

echo "Fin del script exitoso." # Solo se alcanza si no hay errores

Procesamiento de Texto con Herramientas de Línea de Comandos

Bash se integra perfectamente con potentes herramientas como cut, sort, uniq, awk, sed para manipular datos basados en texto.

#!/bin/bash

datos="campo1,campo2,campo3\nlinea2_campo1,linea2_campo2,linea2_campo3"

echo -e "$datos" > data.csv # Guardar datos en un archivo

# Extraer la segunda columna (separada por coma) usando cut
echo "Segunda columna:"
cut -d',' -f2 data.csv

# Ordenar líneas (alfabéticamente por defecto)
echo "Lineas ordenadas:"
echo -e "beta\nalpha\ngamma" | sort

# Procesar texto línea por línea con awk
# Imprimir la primera y tercera columna (separadas por coma)
echo "Awk - Columnas 1 y 3:"
awk -F',' '{ print $1, $3 }' data.csv

# Reemplazar texto con sed (cambiar 'campo' por 'valor')
echo "Sed - Reemplazo:"
echo "$datos" | sed 's/campo/valor/g' # 'g' for global replacement

Nota: Estas herramientas son muy poderosas y tienen sus propios lenguajes o sintaxis complejas.

Comunicación entre Scripts

Los scripts a menudo necesitan interactuar entre sí o con el entorno.

  • Variables de Entorno: Exportar variables para que estén disponibles para scripts hijos.
  • Pipes: Conectar la salida estándar de un script con la entrada estándar de otro.
  • Archivos Temporales: Compartir datos a través de archivos (usando mktemp para crear nombres seguros).
  • Códigos de Salida: Un script padre puede verificar el éxito o fallo de un script hijo usando $?.
#!/bin/bash

# Script Padre (parent.sh)

# Exportar una variable
export VAR_GLOBAL="Hola desde el padre"

echo "Llamando a script hijo..."
./child.sh # Ejecutar el script hijo

echo "El script hijo terminó con código de salida: $?"

# Usando pipes para enviar datos al hijo
echo "Dato para el hijo" | ./child_pipe.sh

echo "Padre terminado."

#!/bin/bash

# Script Hijo (child.sh)

# Acceder a la variable global exportada por el padre
echo "Script hijo: VAR_GLOBAL es '$VAR_GLOBAL'"

# Terminar con un código de salida
exit 5 # Un código de salida diferente de 0

#!/bin/bash

# Script Hijo que lee de stdin (child_pipe.sh)

echo "Script hijo (pipe): Leyendo de stdin..."
read dato_recibido
echo "Script hijo (pipe): Recibido: '$dato_recibido'"

Para probar los ejemplos de comunicación, guarda el primer script como parent.sh, el segundo como child.sh y el tercero como child_pipe.sh. Dales permisos de ejecución (chmod +x parent.sh child.sh child_pipe.sh) y ejecuta ./parent.sh.

Aplicaciones Prácticas de Bash Scripting

Bash scripting es indispensable para muchas tareas en entornos de servidor y desarrollo. Aquí se detallan algunas aplicaciones comunes:

Administración de Servidores

Los administradores de sistemas utilizan Bash a diario para automatizar tareas repetitivas en servidores Linux/Unix.

  • Gestión de Usuarios: Crear, modificar o eliminar usuarios y grupos.
  • Monitorización: Scripts para comprobar el uso de CPU, memoria, disco; verificar si los servicios están activos.
  • Gestión de Logs: Rotar, comprimir o analizar archivos de log del sistema y aplicaciones.
  • Configuración del Sistema: Automatizar la instalación de software, configuración de red, ajustes del firewall.
  • Tareas Programadas: Usar Cron o systemd timers para ejecutar scripts a intervalos regulares.
#!/bin/bash

# Ejemplo: Verificar si un servicio está activo
servicio="apache2" # o "nginx", "mysql", etc.

if systemctl is-active --quiet "$servicio"; then
    echo "El servicio $servicio está activo."
else
    echo "El servicio $servicio NO está activo."
    # Opcional: Reiniciar el servicio
    # sudo systemctl restart "$servicio"
fi

Copias de Seguridad (Backups)

Automatizar copias de seguridad es una de las tareas más críticas para cualquier sistema.

  • Copias Locales: Scripts que comprimen directorios (tar, zip) y los copian a otra ubicación en el mismo servidor o en un almacenamiento local adjunto.
  • Copias Remotas: Usar scp o rsync para transferir copias de seguridad a servidores remotos de forma segura.
  • Backups de Bases de Datos: Scripts que usan herramientas como mysqldump o pg_dump para crear copias de bases de datos.
  • Rotación de Backups: Eliminar copias de seguridad antiguas para liberar espacio.
#!/bin/bash

# Ejemplo: Backup simple de un directorio usando tar y gzip
SOURCE_DIR="/var/www/html" # Directorio a respaldar
BACKUP_DIR="/backup/web" # Directorio de destino
TIMESTAMP=$(date +"%Y%m%d_%H%M%S")
BACKUP_FILE="web_backup_${TIMESTAMP}.tar.gz"

mkdir -p "$BACKUP_DIR"

echo "Creando copia de seguridad de $SOURCE_DIR..."
tar -czf "$BACKUP_DIR/$BACKUP_FILE" "$SOURCE_DIR"

if [ $? -eq 0 ]; then
    echo "Copia de seguridad creada exitosamente: $BACKUP_DIR/$BACKUP_FILE"
else
    echo "Error al crear la copia de seguridad."
fi

Despliegues (Deployments)

Automatizar el proceso de llevar código de desarrollo a producción.

  • Transferencia de Archivos: Copiar código a través de SSH (scp, rsync).
  • Actualización de Dependencias: Ejecutar comandos como npm install, pip install, composer install en el servidor remoto.
  • Reiniciar Servicios: Reiniciar servidores web o de aplicaciones después de un despliegue.
  • Gestión de Versiones: Usar Git para obtener la última versión del código.
  • Rollbacks: Scripts para volver a una versión anterior en caso de problemas.
#!/bin/bash

# Ejemplo: Despliegue simple con rsync
USER="tu_usuario_ssh"
HOST="tu_servidor.com"
SOURCE_PATH="/ruta/local/del/proyecto/"
DEST_PATH="/ruta/remota/del/servidor/web/"

echo "Iniciando despliegue a $HOST..."

# Usar rsync para sincronizar archivos
rsync -avz "$SOURCE_PATH" "$USER@$HOST:$DEST_PATH" --delete

if [ $? -eq 0 ]; then
    echo "Despliegue exitoso."
    # Ejemplo: Opcional, reiniciar un servicio en el servidor remoto
    # ssh "$USER@$HOST" "sudo systemctl restart tu_servicio_app.service"
else
    echo "Error durante el despliegue."
fi

Análisis de Logs

Los archivos de log contienen información vital sobre el comportamiento del sistema y las aplicaciones.

  • Filtrado: Usar grep para encontrar entradas específicas (ej. errores, advertencias, peticiones de una IP).
  • Extracción de Datos: Usar awk o cut para extraer campos específicos (ej. timestamp, IP, URL de logs web).
  • Reportes: Scripts que procesan logs para generar resúmenes, contar ocurrencias, identificar patrones.
  • Monitorización en Tiempo Real: Usar tail -f combinado con otros comandos para ver logs a medida que se escriben.
#!/bin/bash

# Ejemplo: Contar errores en un archivo de log
LOG_FILE="/var/log/syslog" # o "/var/log/apache2/error.log", etc.

if [ ! -f "$LOG_FILE" ]; then
    echo "Error: Archivo de log '$LOG_FILE' no encontrado."
    exit 1
fi

echo "Analizando errores en $LOG_FILE..."

# Contar líneas que contienen la palabra "error" (ignorando mayúsculas/minúsculas)
error_count=$(grep -ic "error" "$LOG_FILE")

echo "Se encontraron $error_count entradas de error."

# Mostrar las últimas 10 líneas que contienen la palabra "error"
echo "Últimos 10 errores:"
grep -i "error" "$LOG_FILE" | tail -n 10

Estas son solo algunas de las muchas formas en que Bash scripting se aplica en escenarios del mundo real, mejorando la eficiencia y la capacidad de automatización.
